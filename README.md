# Chirp Changelog

A polished, one-click release-notes site powered by [Chirp](https://lbliii.github.io/chirp/) and PostgreSQL.

Chirp Changelog gives a small product a public timeline, searchable and tagged archives, stable release permalinks, and a standards-based Atom feed. A generated owner token unlocks a lightweight studio for drafting, publishing, editing, and deleting Markdown release notes. There are no user accounts or external services.

## Run locally

You need Python 3.14 and [uv](https://docs.astral.sh/uv/).

```bash
uv sync
uv run python app.py
```

Open `http://localhost:8000`. In development, the owner token is `changelog-local-admin` unless `CHIRP_ADMIN_TOKEN` is set.

## Deploy on Railway

The Railway template provisions one web service and PostgreSQL, generates the signing and owner secrets, runs migrations before each deployment, and waits for `/ready` before promotion.

Required variables are wired by the template:

- `DATABASE_URL` from Railway PostgreSQL
- `CHIRP_SECRET_KEY` generated by Railway
- `CHANGELOG_ADMIN_TOKEN` generated by Railway
- `RAILWAY_PUBLIC_DOMAIN` supplied automatically by Railway
- `CHIRP_ENV`, `CHIRP_SKIP_MIGRATIONS`, and `PORT`

After deployment, copy `CHANGELOG_ADMIN_TOKEN` from the web service variables and use **Owner access** at the bottom of the timeline rail.

Redis is intentionally not required. Chirp's signed cookie sessions are sufficient for this single-replica starter, and durable releases live in PostgreSQL.

## Product contracts

- Drafts never appear on public pages or in the feed.
- Markdown is sanitized by Chirp before becoming template-safe HTML.
- A release keeps its original slug when its title changes.
- Full-page and HTMX authoring share the same route handlers.
- Atom output is valid XML with absolute entry URLs.
- Database and migration failures stop startup with actionable errors.

## Verify

```bash
uv run ruff check .
uv run ruff format . --check
uv run pytest -q
```

The behavioral suite is traced to [Chirp issue #810](https://github.com/lbliii/chirp/issues/810).

## Eject and customize

This repository is the complete application. Fork it, change `templates/index.html` and `styles.css`, add SQL migrations, or point an existing Railway service at your fork. PostgreSQL contains all durable state; there is no hidden control plane.
